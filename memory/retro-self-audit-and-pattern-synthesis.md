A retro that scans only the work (bugs, domain traps, decisions) and scores candidates one-by-one
misses two things: (a) lessons about the AGENT's own analysis / recommendation failures, and (b) the
recurring PATTERN behind several events (often the single most valuable lesson). Proven once: a first
retro pass surfaced only comfortable TECHNICAL findings and missed the deepest meta-lesson — because
its event list was subject-biased and its pipeline was per-candidate with no synthesis step. Re-running
with a self-lens + a synthesis step immediately surfaced the pattern.

**Why:** the highest-value lesson of a painful session is usually "how I systematically went wrong",
which is both (a) uncomfortable to write and (b) a pattern across events, not a single atom — exactly
what a subject-only, atomic retro cannot see. A "dare to report zero" honesty gate can even become
cover for dodging the uncomfortable self-lesson.

**How to apply:** in any end-of-session retro, add two passes. (1) A SELF lens: "where did the user
correct me, where did I reverse myself, where did I declare done prematurely or relay a claim I hadn't
checked?" (2) A SYNTHESIS step before scoring: "looking across the events, what is the common root
cause / recurring behavioral pattern?" Capture the pattern, not just the atoms.
